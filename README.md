
# Blinkit-Grocery-Sales-Analysis-Power-BI-

# 📌 Brief One Line Summary

An interactive Power BI dashboard analyzing ₹18.59M in grocery sales to uncover channel performance, item popularity, and outlet profitability.

---

## 🔍 Overview

This repository contains a comprehensive **Blinkit Grocery Sales Analysis Dashboard** built using Power BI. The project transforms raw retail transaction data into actionable business insights, helping stakeholders track Key Performance Indicators (KPIs), understand customer purchasing behavior, and optimize inventory across different outlet types.

---

## 🎯 Problem Statement

Managing a massive quick-commerce grocery network requires deep visibility into sales trends. Without a centralized analytics tool, it is difficult to identify:

* Which product categories drive the highest revenue.
* How outlet type, size, and establishment age affect total sales.
* The distribution of consumer preference between low-fat and regular items.

This dashboard solves these challenges by providing real-time visual tracking of sales metrics, item properties, and outlet efficiency.

---

## 📊 Dataset

The dataset used for this analysis includes comprehensive transactional data from Blinkit outlets, capturing key attributes such as:[Blinkit.xlsx](https://github.com/user-attachments/files/30082513/Blinkit.xlsx)


* **Sales Metrics:** Total Sales, Average Sales, Item MRP.
* **Product Attributes:** Item Category (Type), Fat Content (Low Fat vs. Regular).
* **Outlet Details:** Outlet Type, Establishment Year, Location Size.

---

## 🛠️ Tools and Technologies

* **Data Visualization & Analytics:** Power BI Desktop
* **Data Transformation:** Power Query Editor
* **Data Modeling:** DAX (Data Analysis Expressions) for custom KPIs
* **Documentation & Version Control:** Git & GitHub

---

## 🧪 Methods & Workflow

1. **Data Cleaning & ETL:** Imported raw data into Power Query, handled missing values, and standardized categorical data (e.g., product types and outlet categories).
2. **Data Modeling:** Created relational schemas and established necessary DAX measures for core KPIs like `Total Sales`, `Average Sales`, and `Total Orders`.
3. **UI/UX Design:** Formatted charts with a clean, cohesive color palette (Green and Light/Dark Blue) with drop shadows for distinct visual hierarchy.
4. **Interactivity:** Integrated dynamic slicers (Outlet Type filter) to let users drill down into specific store segments.

---

## 💡 Key Insights

Based on the dashboard analysis:

* **Core KPIs:** Generated **₹18.59M in Total Sales** across **8,523 orders** with an Average Item MRP of **₹140.99**.
* **Top Performers:** **Fruits and Vegetables (₹2.8M)** and **Snack Foods (₹2.7M)** are the dominant revenue-generating categories.
* **Channel Efficiency:** **Supermarket Type 1** is the clear winner, contributing a massive **₹7.6M** to the overall sales.
* **Consumer Preferences:** Low-fat items are highly popular, accounting for **63.42% (₹6.95M)** of total fat-content sales compared to Regular fat items (36.58%).
* **Timeline Trend:** A significant sales dip occurred around the late 1990s (near 1998), followed by a steady stabilization between ₹1.1M and ₹1.3M in the 2000s.

## 🏁 Results & Conclusion

The analysis highlights that **Supermarket Type 1** and health-conscious choices (**Low-Fat products**) are major growth drivers for Blinkit. By focusing inventory efforts on fast-moving categories like Fruits, Vegetables, and Snacks while optimizing underperforming outlets (like traditional Grocery Stores at ₹0.2M), the business can significantly maximize its profit margins.
